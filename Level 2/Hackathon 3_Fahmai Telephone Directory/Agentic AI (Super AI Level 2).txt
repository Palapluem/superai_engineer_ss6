How can LLMs solve multi-step tasks?
Workflows are systems where LLMs and tools are orchestrated through predefined code paths.
Use when you can picture the exact flow or steps that AI should go through to solve a problem, or when your app's UX constrains users to a set of tasks
Agents are systems where LLMs dynamically direct their own processes and tool usage, maintaining control over how they accomplish tasks.
Use when you're not sure exactly what task or task parameters you'll give to AI

Context Confusion

Context Confusion is when superfluous content in the context is used by the model to generate a low-quality response.
If you put something in the context the model has to pay attention to it.
https://www.dbreunig.com/2025/06/22/how-contexts-fail-and-how-to-fix-them.html 
There can be too many tools

Context Clash
Context Clash is when you accrue new information and tools in your context that conflicts with other information in the context.
This is a more problematic version of Context Confusion: the bad context here isn’t irrelevant, it directly conflicts with other information in the prompt.

https://www.dbreunig.com/2025/06/22/how-contexts-fail-and-how-to-fix-them.html

In-context Learning (ICL)
ICL: The model can learn from the examples you give it, and will detect and pick up any patterns in the text. 
ICL is highly unpredictable. 
https://ai.stanford.edu/blog/understanding-incontext/ 
https://arxiv.org/abs/2202.12837 

Chat History Effect
In any long conversation, the biggest piece of context you have is your chat history.
It's large and naturally repetitive, forming a strong pattern. 
This triggers in-context learning (ICL) and LLMs start to take the chat history as the narration example to follow. 
This effect is self-reinforcing, which makes it even worse.
ICL is impossible for the model to resist if the pattern is strong enough.
For these reasons, you really really don't want to play with a long chat history, and should trim it past maybe ~30 messages or so.

In-context learning is a fundamental LLM behavior. It's extremely low-level, and powers almost everything in LLMs.

Checklist: “Should I build an agent”
Is the task complex enough?
No → Workflows
Yes → Agents
Is the task valuable enough?
<$0.1 → Workflows
>$1 → Agents
Are all parts of the task doable?
No → Reduce scope
Yes → Agents
What is the cost of error/error discovery?
High -> Read-only/human-in-the-loop
Low -> Agents

Summary
Tool use allows LLM to interact with various applications.
Workflow is a predetermined LLM system.
Agentic AI is an LLM with tools in a loop.
As agents work, context grows. Managing context becomes critical.
Reliability becomes the critical aspect of agents.
Evaluating agents is still tricky.
Human-in-the-loop: Humans should be able to quickly verify agents’ actions.
