โปรเจกต์นี้เกิดจากความสนใจที่อยากจะใช้ Data Science เข้ามาแก้ปัญหา **Resource Allocation** หรือการจัดสรรรถให้ตรงกับความต้องการของผู้โดยสารจริง ๆ ครับ"

### 2. ปัญหาและ Pain Point (Problem & Pain Point) [0:20 - 0:45]
**(Action: ตัดภาพไปที่สไลด์โชว์กราฟเส้นที่ยุ่งเหยิง หรือข้อมูลดิบ)**
"ปัญหาสำคัญคือ ข้อมูลการเดินทางของไทยมักกระจัดกระจายและดูเป็นรายวัน ซึ่งมีความผันผวนสูงมากครับ (Pain Point)
ถ้าเราดูแค่ข้อมูลดิบ เราจะไม่เห็น Pattern ที่แท้จริง ทำให้ผู้ให้บริการเดินรถไม่สามารถวางแผนรับมือช่วงคนแน่น หรือช่วงเทศกาลได้อย่างแม่นยำ
นำไปสู่ปัญหาคลาสสิกคือ **'รถขาดช่วงตอนคนเยอะ และรถโล่งตอนคนน้อย'** ทำให้เสียโอกาสทางเศรษฐกิจและเวลาครับ"

---

### 3. แนวคิดการแก้ปัญหา (Data Science/AI Approach) [0:45 - 1:10]
**(Action: โชว์ Flowchart หรือเครื่องมือ Python logo)**
"ผมจึงเลือกใช้กระบวนการ **Exploratory Data Analysis (EDA)** ผสานกับ **Statistical Methods** มาช่วยครับ
ทำไมถึงเลือกวิธีนี้? เพราะโจทย์คือการทำความเข้าใจพฤติกรรม (Behavior) ไม่ใช่แค่การทำนาย
ผมใช้หลักการทางสถิติ 3 อย่างหลักคือ:
1. **Time-Series Decomposition**: ใช้ Rolling Average เพื่อตัด Noise รายวันออก
2. **Coefficient of Variation (CV)**: เพื่อวัดความนิ่งของแต่ละสาย
3. **Z-Score Anomaly Detection**: เพื่อจับผิดปกติในช่วงเทศกาลโดยอัตโนมัติครับ"

---

### 4. การเตรียมข้อมูล (Data Prep & Analysis) [1:10 - 1:40]
**(Action: โชว์โค้ดส่วน Data Loading / Cleaning สั้นๆ)**
"เริ่มจาก Data Source ที่เป็นไฟล์ CSV ข้อมูลผู้โดยสารปี 2568 และ 2569 มา Combine กัน
**Data Cleaning Step:**
*   ผมจัดการเรื่อง Format วันที่และ mapping ชื่อสายรถไฟฟ้าให้เป็นมาตรฐานเดียวกัน
*   ตรวจสอบ Missing Value และจัดการ Data Type ให้สามารถคำนวณทางคณิตศาสตร์ได้
*   สิ่งที่ท้าทายคือ Feature Engineering ครับ ผมสร้าง Feature ใหม่ เช่น `is_weekend` (วันหยุดสุดสัปดาห์) และ `event_type` (เทศกาล) เพื่อใช้ Cross-check กับปริมาณผู้โดยสารครับ"

---

### 5. Algorithm และการลงมือทำ (Implementation & Coding) [1:40 - 2:15]
**(Action: โชว์หน้าจอ VS Code เลื่อนดูโค้ดไฮไลท์)**
"ในส่วนของ Implementation ผมใช้ Python ผ่าน Library Pandas และ Matplotlib ในการวิเคราะห์ครับ
จุดที่เป็น Highlight คือการเขียน Algorithm **Anomaly Detection** ครับ (ชี้ที่โค้ด Q3 ใน Notebook)
ผมไม่ได้แค่พล็อตกราฟดูด้วยตาเปล่า แต่ผมเขียน Logic คำนวณ **Z-Score**
โดยกำหนด Threshold ที่ **±2.0 SD** เพื่อให้คอมพิวเตอร์ Flag วันที่ผิดปกติออกมาให้เอง
ทำให้เราเจอ Insight สำคัญว่า 'วันเปิดทำงานวันแรกหลังสงกรานต์' คือวันที่ระบบรับภาระหนักที่สุด (Spike Demand) ไม่ใช่วันทำงานปกติครับ"

---

### 6. สรุปผลลัพธ์ (Conclusion & Insight) [2:15 - 2:35]
**(Action: โชว์ Dashboard กราฟ 3 รูปที่สวยงาม)**
"ผลลัพธ์ที่ได้สรุปออกมาเป็น 3 Strategic Insights ครับ:
1. **Market Share:** 2 สายหลัก (BTS/สีน้ำเงิน) ครองตลาด 80% แต่สายชานเมือง (ARL/แดง) โตแรงที่สุด
2. **Behavior:** สายสีม่วงและสายสีแดงมีความผันผวนสูง (High Volatility) ต้องจัดการรถแบบ Dynamic ถึงจะคุ้มทุน
3. **Holiday Effect:** เทศกาลทำให้คนหายไป 40% แต่จะดีดกลับมาทันทีหลังจบเทศกาล ซึ่งเป็นจุดที่ต้องเตรียมรถเสริมครับ"
